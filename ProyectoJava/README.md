## Organización del repositorio

Durante este semestre trabajamos con este repositorio online en GitHub y nos organizamos de la siguiente manera:

- **Rama `main`**  
  - Se mantuvo completamente vacía, ya que consideramos que realizar *merge* desde todas las ramas sería engorroso.  
  - Los temas trabajados son similares entre lenguajes y comparten la misma numeración, por lo que no era necesario fusionarlas.

- **Rama `Python`**  
  - Contiene todos los ejercicios numerados y con el título correspondiente de la plataforma.

- **Rama `Java`**  
  - Mismo formato que la rama Python.

- **Rama `JavaScript`**  
  - También con los ejercicios numerados y titulados.

> **Nota:** No consideramos oportuno trabajar con GitHub para mantener la versión final del proyecto grupal, ya que en múltiples ocasiones no realizaba una copia fiel del trabajo.

## Ramas adicionales

Además de las ramas principales, contamos con dos ramas auxiliares:

- **Rama `TAREAS-EN-VIVO`**  
  Utilizada para realizar los commits de manera ordenada y facilitar la metodología de entrega.

- **Rama `PROYECTO-JAVA`**  
  Sirvió como base para el desarrollo del proyecto semestral.  
  Cada integrante fue realizando commits con sus modificaciones a medida que avanzaba el proceso.
